## 1. Purpose

This policy defines acceptable use of {{company_name}}'s information systems, devices, and network resources.

## 2. Scope

All devices, accounts, and network access used for company business, including personal devices used for work.

## 3. Policy Statements

### 3.1 Device Security
- Full-disk encryption is enabled on all devices (FileVault/BitLocker).
- Firewall is enabled on all devices.
- Automatic OS and software updates are enabled.
- Screen lock activates after 5 minutes of inactivity.
- Antivirus/endpoint protection is active and updated.

### 3.2 Network Security
- VPN or secure connection is used when accessing company resources from public networks.
- Company credentials are never used on untrusted networks without VPN.

### 3.3 Account Security
- Unique, strong passwords for all accounts (12+ characters).
- Password manager is used for credential storage.
- MFA is enabled on all accounts (see Access Control Policy, POL-002).
- Credentials are never shared, stored in plaintext, or committed to code.

### 3.4 Data Handling
- Company data is stored in authorized systems only (not personal cloud storage).
- Confidential data is handled per the Data Classification Policy (POL-008).
- Company data is not transmitted via unencrypted channels.

### 3.5 Software
- Only authorized software is installed on work devices.
- Software is downloaded from official sources only.
- Browser extensions are reviewed for security implications.

## 4. Procedures

### 4.1 Device Setup Checklist
1. Enable full-disk encryption
2. Enable firewall
3. Enable automatic updates
4. Configure screen lock (5 min timeout)
5. Install endpoint protection
6. Configure password manager
7. Enable MFA on all accounts
8. Document device in asset inventory

## 5. Exceptions

Use of personal devices for company work requires compliance with all device security requirements in this policy.

## 6. Enforcement

Non-compliant devices may have access revoked until security requirements are met.
